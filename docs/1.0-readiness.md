# TypeWhisper 1.0 就绪

TypeWhisper `1.0` 是面向 macOS 的稳定直链下载版本。Mac App Store 完全不在此版本定义的范围内。在 `1.0` 之前，不会新增主要产品功能面，聚焦于可靠性、清晰的产品边界、升级安全性和快速的支持诊断。

## 目标受众

- 需要系统级听写并可自主选择引擎的 macOS 用户
- 需要在本地或通过 API 转录音视频文件的用户
- 需要提示词处理、Profile 和本地自动化的高级用户

## 官方支持的核心功能

- 通过全局快捷键进行系统级听写和文本插入
- 文件转录，包括批量处理和导出
- 提示词处理，含内置预设和自定义动作
- 基于应用和 URL 的 Profile 控制
- 历史记录、词典和代码片段
- 捆绑的默认集成和默认插件

## 1.0 中的高级功能面

这些功能面仍属于 `1.0` 的一部分，但被定位为高级或自动化功能面：

- 本地 HTTP API（`/v1/*` 路径下）
- `typewhisper` CLI
- 插件 SDK 和插件清单
- Widgets
- 监视文件夹

## 1.0 前的非目标

- 不发布 Mac App Store 版本，不设 App Store 专用门槛
- 不引入新的主要引擎、插件类或公共 API 系列
- 不对已文档化的 HTTP API、CLI 或插件 SDK 做破坏性更改
- `1.0` 不要求广泛的 UI 自动化

## 1.x 稳定性契约

### HTTP API

- `/v1/*` 下已文档化的端点在 `1.x` 期间保持稳定。
- 响应字段在 `2.0` 之前不得删除，须先废弃。
- API 仅限回环地址，默认禁用，用于本地自动化。

### CLI

- `typewhisper status`
- `typewhisper models`
- `typewhisper transcribe`
- 参数：`--port`、`--json`、`--language`、`--task`、`--translate-to`

### 插件 SDK

- `manifest.json`
- `TypeWhisperPlugin`
- `PostProcessorPlugin`
- `LLMProviderPlugin`
- `TranscriptionEnginePlugin`
- `ActionPlugin`
- `HostServices`

## 发布门槛

只有满足以下所有条件后，才能打 `1.0.0` 标签：

- `xcodebuild test`（应用 scheme）通过。
- `swift test --package-path TypeWhisperPluginSDK` 通过。
- 应用 Release 构建通过。
- 无第一方构建警告。
- 插件清单验证成功。
- README、安全指南和支持矩阵为最新。
- `1.0.0-rc1` 构建在真实机器上运行多日无 P0/P1 级阻塞问题。
- 默认频道保持为 `stable`；`release-candidate` 和 `daily` 作为 Sparkle 频道存在，用于预览构建。
- `1.0.0-rc*` 和每日构建以 GitHub 预发布版本分发，仅在各自频道上出现在共享 Sparkle appcast 中，不更新 Homebrew。

## 打标签前的手动冒烟检查

- 在干净机器上全新安装
- 权限流程：麦克风、辅助功能、撤销后的恢复
- 首次成功听写
- 文件转录
- 应用内提示词动作
- 历史编辑/导出
- 应用和 URL 的 Profile 匹配
- 插件启用/禁用
- CLI 对接运行中的本地服务器
- HTTP API `status`、`models`、`transcribe`
- 从 `0.14.x` 升级，历史记录、Profile、词典、代码片段、快捷键和插件状态完整保留

## 支持与诊断

- 支持请求应始终附上错误日志中导出的 JSON 诊断信息。
- 导出内容包括应用版本、macOS 版本、API 状态、已启用插件和隐私安全的设置元数据。
- 导出内容不包含 API 密钥、音频数据和转录历史。
